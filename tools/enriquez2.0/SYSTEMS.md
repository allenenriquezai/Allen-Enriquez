# Enriquez2.0 — Systems Index

Master index of every tool, service, and automation in the repo. **Loaded on demand only** — Claude/Allen pulls this when answering "how does X work." Not loaded into every session.

> **Rule:** every tool with non-trivial moving parts gets an `OVERVIEW.md` next to its code (data flow, entry points, inputs/outputs, failure modes, dependencies, last-verified sha). README.md is for install/deploy ceremony; OVERVIEW.md is for "how does this thing actually work at 11pm three months from now."

## Skills + workflows

- **Workflows** (SOPs): `projects/eps/workflows/`, `projects/personal/workflows/`
- **Routing tables:** `CLAUDE.md` (top-level), `projects/*/CONTEXT.md` (per-domain)

## How OVERVIEW.md gets created

1. Hand-written. Never auto-generated by Claude (drift + lies).
2. Strict shape: `What it does` → `Data flow` (mermaid) → `Entry points` → `Inputs` → `Outputs` → `Failure modes` → `Dependencies` → `Last verified <date> @ <sha>`.
3. When the tool's code changes substantially, the human re-verifies and updates the sha. Future `enriquez2.0/scripts/staleness.ts` will flag drift via `git diff <sha>..HEAD -- <listed-entry-points>`.

## Update protocol

- **New tool added to repo** → write `<tool>/OVERVIEW.md` before merging the PR. Add row here.
- **Major refactor of existing tool** → bump `Last verified` sha after re-checking the doc against code.
- **Tool retired** → move row to "Inactive / archive" section. Don't delete the doc.
