## Principles (seed)

- One repo per person, domain folders inside (`projects/eps/`, `projects/personal/`, `tools/<domain>/`).
- Skills are entry points, not agent spawners. Read workflows, do work in main thread.
- Token discipline — read only what's needed for the task. CONTEXT.md + one workflow.
- Atomic commits. Each commit ships one verifiable thing.
- Fail loud, never silent. No fabricated data.
- Local over API. Batch over real-time. Cost ≈ $0 default.
