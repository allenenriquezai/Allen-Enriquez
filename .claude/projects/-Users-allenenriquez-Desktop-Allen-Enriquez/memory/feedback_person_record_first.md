---
name: Person Record First
description: Emails and activities always go on the person record in Pipedrive, not the project
type: feedback
---

Always use the person record for emails and activities. Projects in Pipedrive don't support sending emails — only leads and deals do via the person.

**Why:** Person is the main source of truth for contact in Pipedrive. Projects are for tracking service delivery stages, not communication.

**How to apply:** When creating activities or sending emails, always attach to person_id (and optionally deal_id). Never try to send from a project record.
